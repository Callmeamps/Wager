Contributing Guide

Thanks for your interest in Wager! Here's how to get started.

Development Setup

git clone https://github.com/your-org/wager.git
cd wager
npm ci
npm run dev

Open http://localhost:4317 in your browser.

Project Structure

wager/
├── src/
│   ├── lib/
│   │   ├── db.ts              # Prisma client
│   │   ├── validations.ts     # Zod schemas
│   │   └── server/            # Server-only libs
│   │       ├── auth.ts        # JWT, bcrypt helpers
│   │       ├── rate-limit.ts  # Rate limiter
│   │       └── require-auth.ts
│   ├── routes/
│   │   ├── api/v1/            # REST endpoints
│   │   ├── login/             # Login page
│   │   ├── register/          # Register page
│   │   ├── wagers/            # Wager list & detail
│   │   └── profile/           # User profile
│   └── hooks.server.ts        # Auth & rate limit middleware
├── prisma/
│   └── schema.prisma          # Database schema
├── scripts/
│   ├── install.sh             # One-command installer
│   ├── deploy.sh              # Deployment guide
│   └── seed.ts                # Demo data seeder
└── docs/                      # Documentation

Coding Standards

  • TypeScript — Strict mode, no any
  • Svelte 5 — Use runes ($state, $derived, $props)
  • Files — kebab-case (wager-card.svelte)
  • Components — PascalCase (WagerCard)
  • Functions/vars — camelCase

Pull Request Process

  1. Create a feature branch from main
  2. Make changes, add tests when applicable
  3. Run npm run check — must pass with 0 errors
  4. Run npm run build — must succeed
  5. Commit with Conventional Commits format
  6. Push and open a PR against main
  7. Wait for CI to pass
  8. Merge after review

Commit Format

type(scope): subject

  • feat(wager): add proof submission with file upload
  • fix(auth): handle duplicate email registration
  • docs: update README with Docker instructions

Running Tests

# Type check (no runtime tests yet — PRs welcome!)
npm run check

We welcome contributions to add unit, integration, and e2e test suites.

Need Ideas?

Check the open issues for feature requests and bug reports. Good starter projects:

  • Add unit tests for validation schemas
  • Add e2e tests with Playwright
  • Email notifications (password reset, reminders)
  • Comments system on wagers
  • File upload support for proof images
  • PWA offline support
