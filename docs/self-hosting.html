Self-Hosting Guide

Deploy Wager on your own infrastructure.

Environment Variables

VariableRequiredDescription
DATABASE_URLYesSQLite: file:./prisma/data/wagers.db or MySQL/Dolt: mysql://user:pass@host:3306/db
JWT_SECRETYesAt least 32 random chars. Generate: openssl rand -base64 32
PORTNoServer port (default 4317)
NODE_ENVNodevelopment or production
ALLOWED_ORIGINSNoComma-separated CORS origins

Option A: Docker (Recommended)

git clone https://github.com/your-org/wager.git
cd wager
docker-compose build
docker-compose up -d
# App at http://localhost:4317

For production, set environment variables in docker-compose.yml or pass them via docker-compose run -e.

Option B: Railway

  1. Push code to a GitHub repository
  2. Create a new Railway service → Deploy from GitHub repo
  3. Set build command: npm run build
  4. Set start command: npx prisma migrate deploy && npm start
  5. Add environment variables (DATABASE_URL, JWT_SECRET, NODE_ENV=production)
  6. For Dolt/MySQL, provision a Railway MySQL plugin or external Dolt instance

Option C: Render

  1. Push code to GitHub
  2. Create a new Web Service → connect repo
  3. Runtime: Node
  4. Build command: npm ci && npm run build
  5. Start command: npx prisma migrate deploy && npm start
  6. Add environment variables

Option D: Vercel / Netlify

Important: Serverless platforms have ephemeral filesystems. SQLite file databases will not persist between deployments. Use an external MySQL/Dolt database for production.

The app uses @sveltejs/adapter-auto which auto-detects the platform. Deploy via Vercel CLI or Git push.

Database Persistence

For SQLite, the database file lives at prisma/data/wagers.db. Mount this path as a persistent volume in Docker, or use Railway's volume service.

For Dolt/MySQL, use a managed database service or run Dolt as a sidecar container.

Health Check

The GET /api/v1/wagers endpoint returns a JSON response when the server is healthy. Use this for load balancer health checks.
