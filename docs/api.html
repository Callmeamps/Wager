API Reference

All endpoints are prefixed with /api/v1. Base URL: https://wager.example.com/api/v1 (production) or http://localhost:4317/api/v1 (dev).

Authentication: Protected routes require an Authorization: Bearer <token> header OR a session cookie (httpOnly). The token is a JWT valid for 15 minutes. Refresh with POST /auth/refresh.

Error format:

{
  "error": true,
  "message": "Human-readable error message",
  "code": "VALIDATION_ERROR", // optional
  "details": {} // optional additional context
}

Authentication Endpoints

POST /auth/register

Create new user account.

Request Body

{
  "email": "user@example.com",
  "password": "securePassword123",
  "name": "Optional Name"
}

Responses

  • 201 Created: { "user": { "id": "...", "email": "...", "name": "..." } }
  • 400 Bad Request: Validation error (email invalid, password too short)
  • 409 Conflict: Email already exists
Public

POST /auth/login

Authenticate, receive JWT + refresh token (set as cookies).

Request Body

{
  "email": "user@example.com",
  "password": "securePassword123"
}

Responses

  • 200 OK: { "user": {...}, "accessToken": "jwt..." }; cookie: refresh_token=...
  • 401 Unauthorized: Invalid credentials
Public

POST /auth/logout

Invalidate session. Clears session cookie.

Responses

  • 204 No Content
Auth

POST /auth/refresh

Exchange refresh token cookie for new access token.

Responses

  • 200 OK: { "accessToken": "new-jwt..." }
  • 401 Unauthorized: Invalid/expired refresh token
Auth

Wager Endpoints

GET /wagers

List wagers with filters.

Query Parameters

statusFilter by status: pending, submitted, voting, resolved, cancelled
createdByFilter by user ID
participantFilter by participant user ID
limitNumber of results (default 20, max 100)
offsetPagination offset (default 0)
sortField to sort by: createdAt, deadline (default: createdAt desc)

Responses

  • 200 OK: { "wagers": [ {...}, ... ], "total": 42, "limit": 20, "offset": 0 }
Protected

POST /wagers

Create new wager.

Request Body

{
  "title": "Run 5km under 25min",
  "description": "We both run 5km, faster time wins",
  "stake": {
    "amount": "20.00", // string to avoid float issues, in stakeCurrency units
    "currency": "USD"
  },
  "participants": ["user-id-1", "user-id-2"], // must include self
  "deadline": "2025-06-30T23:59:59Z" // ISO 8601, optional
}

Responses

  • 201 Created: { "wager": { ...full wager object... } }
  • 400: Validation failed
  • 401: Unauthorized
Protected

GET /wagers/:id

Get wager by ID, including proofs, votes, participants.

Responses

  • 200 OK: Full wager object
  • 404 Not Found: Wager doesn't exist
Protected

PATCH /wagers/:id

Update wager (only creator can edit). Partial update.

Request Body

{
  "title": "Updated title", // optional
  "description": "Updated desc",
  "deadline": "2025-07-01T00:00:00Z",
  "status": "CANCELLED" // only allowed values: CANCELLED
}

Responses

  • 200 OK: Updated wager
  • 403 Forbidden: Not creator
  • 404: Not found
Protected

DELETE /wagers/:id

Cancel wager (soft delete). Sets status = CANCELLED.

Responses

  • 204 No Content
  • 403 Forbidden: Not creator
  • 404: Not found
Protected

POST /wagers/:id/participants

Add participant to wager (creator only).

Request Body

{
  "userId": "user-id-to-add"
}

Responses

  • 200 OK: Updated wager
  • 403: Not creator
  • 409: Already participant
Protected

Proof Endpoints

GET /wagers/:id/proofs

List all proofs for a wager.

Responses

  • 200 OK: { "proofs": [ {"id":"...", "user":{...}, "link":"...", "createdAt":"..."}, ... ] }
  • 404: Wager not found
Protected

POST /wagers/:id/proofs

Submit proof for current user.

Request Body (multipart or JSON)

{
  "link": "https://example.com/proof.jpg", // required
  "evidence": "base64-encoded..." // optional for file uploads
}

Responses

  • 201 Created: { "proof": {...} }
  • 400: Not a participant, or wager not in correct status
  • 404: Wager not found
Protected

DELETE /proofs/:id

Delete proof. Only proof owner or wager creator can delete.

Responses

  • 204 No Content
  • 403: Not authorized
  • 404: Proof not found
Protected

Vote Endpoints

GET /wagers/:id/votes

List all votes for a wager.

Responses

  • 200 OK: { "votes": [ {"id":"...", "voter":{...}, "isValid":true, "comment":"...", "createdAt":"..."}, ... ] }
Protected

POST /wagers/:id/votes

Cast vote. Participant only; one vote per wager.

Request Body

{
  "isValid": true, // boolean
  "comment": "Optional reasoning" // optional
}

Responses

  • 201 Created: { "vote": {...} }
  • 400: Already voted or not participant
  • 404: Wager not found
Protected

DELETE /votes/:id

Retract vote. Only voter can delete.

Responses

  • 204 No Content
  • 403: Not vote owner
  • 404: Vote not found
Protected

Real-time Stream

GET /wagers/stream

Server-Sent Events stream for live wager updates.

Query Parameters

sinceISO timestamp, only send events after this

Event Types

event: wager_created
data: {"id":"...","title":"...","status":"PENDING",...}

event: proof_added
data: {"wagerId":"...","proofId":"...","userId":"..."}

event: vote_cast
data: {"wagerId":"...","voteId":"...","voterId":"...","isValid":true}

event: wager_updated
data: {"id":"...","status":"RESOLVED","resolution":"WON"}

event: heartbeat
data: {"timestamp":"..."}

Client reconnects automatically on disconnect.

Protected
